#+TITLE: CCL downstream patch ledger
#+AUTHOR: Lukáš Hozda

This directory holds the byte-stable CCL patch mirrors used by cclsh build
attestation. The editable source history lives in the
[[https://github.com/luciusmagn/ccl][LuciusMagn CCL fork]]. The project upstream is
[[https://github.com/Clozure/ccl][Clozure CL]].

The supported downstream line starts at the peeled CCL =v1.13= commit:

#+begin_example
c994217adc56b3f8a564526cee4695893ac84d86
#+end_example

The two downstream commits form a linear branch named =cclsh-v1.13=:

| Role | Commit |
|------+--------|
| CCL v1.13 base | =c994217adc56b3f8a564526cee4695893ac84d86= |
| Linux xstate backport | =a293f4460fadc5da420eec0739e61622e847aed2= |
| cclsh argument boundary and final revision | =579c87300ee632af99182276f2ad40e1c38c5d0a= |

Manual builds clone the fork and detach at the final revision. Nix builds
fetch immutable diffs for the two exact fork commits and apply them to CCL
1.13. Neither path follows a mutable branch at build time.

The local =.patch= files remain in this repository because login-build
attestation hashes their exact bytes. They are mirrors of the changes carried
by the fork, not an independent place to revise those changes. A source change
starts as a focused fork commit. The corresponding local mirror, Nix source,
documentation, tests and attestation expectations then change together.
Cosmetic regeneration of a local mirror is inappropriate because it changes
the attested bytes without changing runtime behavior.

Both mirrors originated in cclsh commit
[[https://github.com/luciusmagn/cclsh/commit/6a9d211fa956cb6a1da5f68d40535f7773c129ff][=6a9d211fa956cb6a1da5f68d40535f7773c129ff=]], =Attest patched
login builds=.

At final fork revision =579c87300ee632af99182276f2ad40e1c38c5d0a=,
=git apply --reverse --check= accepts each local mirror. This records that the
attested downstream changes are present without requiring the fork's commit
serialization and the local attestation files to have identical bytes.

The patch validates the Linux x86-64 extended floating-point state metadata,
copies a complete aligned xstate frame when CCL enables AVX-state copying, and
falls back to the ordinary floating-point frame when validation fails. It is a
modern, bounded implementation informed by the earlier upstream diagnosis,
not a verbatim restoration of the historical change.

- Related upstream issue:
  [[https://github.com/Clozure/ccl/issues/177][Clozure CL issue 177]]

The CCL kernel normally continues recognizing kernel options throughout the
argument vector. A saved cclsh image needs =-c=, script paths and every later
word to reach its own dispatcher unchanged. This patch gives an executable
whose basename is =cclsh= prefix-only kernel option parsing, including the
login-shell basename =-cclsh=.

The focused downstream checks are =scripts/verify-argument-boundary=,
=tests/installed-image.sh= and the script argument cases in
=tests/integration.lisp=. They are included in the release validation sequence
above.

- Classification: downstream-only as written
- Upstream topic branch: none
- Pull request: none submitted
- Upstream merge commit: none
- First upstream release containing an equivalent facility: none
- Upstream design requirement: replace the cclsh basename special case with a
  generic, documented application argument boundary acceptable to CCL
  maintainers
- CCLSH retirement commit: none
- Retirement condition: stock CCL exposes an equivalent generic boundary,
  cclsh adopts it, all argument and installed-image checks pass, and the local
  mirror, immutable Nix diff and attestation entry are removed together

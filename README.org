#+TITLE: cclsh
#+AUTHOR: Lukáš Hozda

A system shell running inside Clozure CL. Every command line is either a
Unix command or a Lisp form, evaluated in a live CCL image.

* Requirements

- Clozure CL (=ccl= in PATH)
- Git (verifies the pinned Clinedi checkout)
- Quicklisp (=~/quicklisp/setup.lisp= by default)
- Clinedi at the revision recorded in =dependencies.lock=, registered as a
  Quicklisp local project
- starship (prompt rendering, with a plain fallback when missing)
- zoxide (optional directory ranking; =fzf= is also needed for =zi=)
- Linux (terminal control talks to libc termios/ioctl directly)

Clinedi supplies the Unicode-aware terminal line editor. For sibling checkouts
below =~/common-lisp/=, add that parent to Quicklisp before registering local
projects, for example in =~/quicklisp/local-init/common-lisp.lisp=:

#+begin_src lisp
(in-package #:quicklisp-client)
(pushnew #P"/root/common-lisp/"
         *local-project-directories*
         :test #'equal)
#+end_src

Quicklisp is also a build dependency: every saved cclsh image includes it,
so =(ql:quickload ...)= works out of the box. Set
=CCLSH_QUICKLISP_SETUP= when its =setup.lisp= is installed somewhere
other than =~/quicklisp/setup.lisp=. The build stops before saving an image if
Quicklisp is unavailable, Clinedi resolves outside its registered local
checkout, or the checkout is dirty or differs from =dependencies.lock=.

* Building

#+begin_src sh
scripts/build
#+end_src

This produces a matched =cclsh= kernel executable and =cclsh.image=
heap image in the repository root. Keeping the image adjacent instead
of appending it to the kernel avoids an intermittent CCL 1.13 image
resume failure on current Linux. Alternatively, from a running CCL:

#+begin_src lisp
(asdf:load-system "cclsh")
(cclsh:main)
#+end_src

Run the regression checks with:

#+begin_src sh
scripts/check
#+end_src

After process, pipeline or terminal changes, rebuild the saved image and
run the PTY integration checks too:

#+begin_src sh
scripts/build
scripts/integration-check
#+end_src

* Behavior

- A line starting with =(= is evaluated as Lisp in the =cclsh-user=
  package, values are printed like a REPL.
- Otherwise the first word resolves to a shell command: a symbol whose
  value is a =command= instance runs as a builtin, anything else is
  looked up in =PATH= and run as an external program.
- When command lookup misses, a line containing only an explicit directory
  path changes into it without spelling =cd=: =..=, =./src=, =src/= and
  =~/work=. Builtins and executables still win, while a bare name such as
  =src= remains a command lookup.
- A lone word that names a bound variable, a keyword or a number
  evaluates REPL style when no command matches: =*last-status*= alone
  just prints its value.
- =~=, =$VAR=, =${VAR}= and =*=/=?= globs expand in bare words. Quoting
  with ="..."= (variables expand) and ='...'= (literal) is supported.
- Parens substitute Lisp values into commands: =echo (*balls*)= prints
  the variable, =echo (+ 1 2)= prints 3, lists splice, NIL vanishes.
- Unfinished input continues on the next line: unbalanced Lisp forms,
  open strings, and trailing backslashes.
- Undefined functions are reported with suggestions before argument
  evaluation can obscure them.
- =~/.config/cclsh/startup.lisp= is loaded on startup, history persists
  in =~/.config/cclsh/history= (=XDG_CONFIG_HOME= is respected).
- External commands run as the terminal's foreground process group, so
  interactive programs work and Ctrl-C reaches the child, not the
  shell. Errors and Ctrl-C never drop you into the CCL debugger.
- Job control: =command &= starts a background job, Ctrl-Z stops the
  foreground command or pipeline, =jobs=, =fg= and =bg= manage them,
  and finished background jobs are announced before the next prompt.
  =exit= warns once about stopped jobs.
- =cclsh -c command=, script files and shebang scripts run without
  user state, which keeps ssh, scp, rsync and git working when cclsh
  is a login shell. =CCLSH_SAFE=1= skips startup.lisp and history.
- Environment variables are lispy: =(setenv 'editor "vim")=,
  =(getenv :home)=, =(setf (env 'port) 8000)=, plus =export= and
  =unset= builtins.
- =(zoxide-setup)= enables =z=, =zi= and automatic recording of every
  successful =cd=. It is safe to call once from =startup.lisp= and does
  nothing destructive when zoxide is unavailable.

* Lisp helpers

#+begin_src lisp
;; Pipelines, sequencing, conditional chains, redirection
(cmd git "status")                ; one command, mid-Lisp
(pipe (ls "-la") (grep "lisp"))   ; ls -la | grep lisp
(seq (make "clean") (make))       ; make clean ; make
(all (make) (make "install"))     ; make && make install
(any (probe) (fallback))          ; probe || fallback
(pipe (make) (to "build.log"))    ; make > build.log, append-to appends
(pipe (from "in.txt") (wc "-l"))  ; wc -l < in.txt
(pipe (make) (error-to "err.log")) ; make 2> err.log
(capture (make) (merge-error))    ; both streams, like 2>&1 in $(...)
(capture (git "rev-parse" "HEAD")) ; $(git rev-parse HEAD) as a string

;; Run a program directly, returns the exit status
(run "git" "status")

;; Define your own commands (usable from the command line and Lisp)
(defcommand gs ()
  "Short git status."
  (run "git" "status" "--short"))

;; Optional ranked directory navigation
(zoxide-setup)
;; z project       ; query zoxide and cd to the result
;; zi              ; choose interactively through fzf
#+end_src

=*last-status*= holds the exit status of the last command.

* Line editor

The interactive editor is provided by Clinedi. Arrow keys navigate history and
the cursor. As you type, the newest
matching history entry appears after the cursor in dim text; Right or
C-f at the end accepts it. Multi-line history entries retain their
newlines. Alt-Enter, or Shift-Enter in terminals that report it, inserts an
explicit newline without submitting. Tab completes command names plus
directories at a slash-free command position, file paths in path-shaped
command words and arguments, and Lisp symbols. Command-position directories
keep the trailing =/= that makes them implicit =cd= paths. Supported keys:
C-a/C-e (home/end), Ctrl-Left/Right (move by
words), C-k/C-u (kill to end/whole line), C-w or C-Backspace/C-h (kill word),
C-l (clear screen), C-d (EOF on an empty line), C-c (abort line). Input is
highlighted with the standard 16 terminal colors: known commands green,
builtins and valid implicit directory paths cyan, unknown commands red,
strings yellow, comments and parens dim, keywords magenta, numbers cyan.
Output that ends without a newline gets a reverse video ⏎ marker, fish style.
The =help= builtin prints the built-in manual (=help substitution= etc.);
=docs/guide.org= is the long form, including login shell setup.

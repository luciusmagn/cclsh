#+TITLE: cclsh
#+AUTHOR: Lukáš Hozda

A system shell running inside Clozure CL. Every command line is either a
Unix command or a Lisp form, evaluated in a live CCL image.

* Requirements

- Clozure CL (=ccl= in PATH)
- starship (prompt rendering, with a plain fallback when missing)
- Linux (terminal control talks to libc termios/ioctl directly)

The shell code itself has zero dependencies beyond CCL and ASDF. When
=~/quicklisp/setup.lisp= exists at build time, Quicklisp is baked into
the produced image so =(ql:quickload ...)= works out of the box; in an
image built without it, =(quicklisp-setup)= loads or installs it.

* Building

#+begin_src sh
scripts/build
#+end_src

This produces a standalone =cclsh= binary in the repository root via
=ccl:save-application=. Alternatively, from a running CCL:

#+begin_src lisp
(asdf:load-system "cclsh")
(cclsh:main)
#+end_src

* Behavior

- A line starting with =(= is evaluated as Lisp in the =cclsh-user=
  package, values are printed like a REPL.
- Otherwise the first word resolves to a shell command: a symbol whose
  value is a =command= instance runs as a builtin, anything else is
  looked up in =PATH= and run as an external program.
- A lone word that names a bound variable, a keyword or a number
  evaluates REPL style when no command matches: =*last-status*= alone
  just prints its value.
- =~=, =$VAR=, =${VAR}= and =*=/=?= globs expand in bare words. Quoting
  with ="..."= (variables expand) and ='...'= (literal) is supported.
- Parens substitute Lisp values into commands: =echo (*balls*)= prints
  the variable, =echo (+ 1 2)= prints 3, lists splice, NIL vanishes.
- Unfinished input continues on the next line: unbalanced Lisp forms,
  open strings, and trailing backslashes.
- Undefined functions are reported with suggestions before argument
  evaluation can obscure them.
- =~/.config/cclsh/startup.lisp= is loaded on startup, history persists
  in =~/.config/cclsh/history= (=XDG_CONFIG_HOME= is respected).
- External commands run as the terminal's foreground process group, so
  interactive programs work and Ctrl-C reaches the child, not the
  shell. Errors and Ctrl-C never drop you into the CCL debugger.
- Job control: =command &= starts a background job, Ctrl-Z stops the
  foreground command or pipeline, =jobs=, =fg= and =bg= manage them,
  and finished background jobs are announced before the next prompt.
  =exit= warns once about stopped jobs.
- =cclsh -c command=, script files and shebang scripts run without
  user state, which keeps ssh, scp, rsync and git working when cclsh
  is a login shell. =CCLSH_SAFE=1= skips startup.lisp and history.
- Environment variables are lispy: =(setenv 'editor "vim")=,
  =(getenv :home)=, =(setf (env 'port) 8000)=, plus =export= and
  =unset= builtins.

* Lisp helpers

#+begin_src lisp
;; Pipelines, sequencing, conditional chains, redirection
(cmd git "status")                ; one command, mid-Lisp
(pipe (ls "-la") (grep "lisp"))   ; ls -la | grep lisp
(seq (make "clean") (make))       ; make clean ; make
(all (make) (make "install"))     ; make && make install
(any (probe) (fallback))          ; probe || fallback
(pipe (make) (to "build.log"))    ; make > build.log, append-to appends
(pipe (from "in.txt") (wc "-l"))  ; wc -l < in.txt
(pipe (make) (error-to "err.log")) ; make 2> err.log
(capture (make) (merge-error))    ; both streams, like 2>&1 in $(...)
(capture (git "rev-parse" "HEAD")) ; $(git rev-parse HEAD) as a string

;; Run a program directly, returns the exit status
(run "git" "status")

;; Define your own commands (usable from the command line and Lisp)
(defcommand gs ()
  "Short git status."
  (run "git" "status" "--short"))
#+end_src

=*last-status*= holds the exit status of the last command.

* Line editor

Arrow keys navigate history and the cursor. Tab completes command
names, file paths and Lisp symbols. Supported keys: C-a/C-e
(home/end), C-k/C-u (kill to end/whole line), C-w (kill word), C-l
(clear screen), C-d (EOF on an empty line), C-c (abort line). Input is
highlighted with the standard 16 terminal colors: known commands green,
builtins cyan, unknown commands red, strings yellow, comments and parens
dim, keywords magenta, numbers cyan. Output that ends without a newline
gets a reverse video ⏎ marker, fish style. The =help= builtin prints
the built-in manual (=help substitution= etc.); =docs/guide.org= is the
long form, including login shell setup.

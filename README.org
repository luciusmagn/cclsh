#+TITLE: cclsh
#+AUTHOR: Lukáš Hozda

A system shell running inside Clozure CL. Every command line is either a
Unix command or a Lisp form, evaluated in a live CCL image.

This is not a POSIX shell. Command strings, scripts, pipelines and redirection
use cclsh syntax. Try it as an ordinary shell before considering login-shell
use.

* Installation

** Nix flake (recommended)

The reproducible package supports Linux x86-64 and needs only Nix with flakes
enabled:

#+begin_src sh
nix run github:luciusmagn/cclsh -- --version
nix profile install github:luciusmagn/cclsh
cclsh --version
#+end_src

From a checkout, use =nix run .= or =nix profile install .#cclsh=. The flake
rebuilds CCL 1.13 with both required kernel patches, fetches the exact Clinedi
revision in =dependencies.lock=, and includes a pinned Quicklisp client and
distribution index. =nix flake check= builds the complete package and checks
its command boundary and installed Quicklisp state.

A packaged image uses an existing =~/quicklisp= when one is present. Otherwise
it initializes a writable tree under
=${XDG_DATA_HOME:-$HOME/.local/share}/cclsh/quicklisp= on first use. Set
=CCLSH_QUICKLISP_HOME= to choose another location. Quicklisp downloads and
compiled files use that writable tree rather than the read-only Nix store.
An explicit target that already exists must contain =setup.lisp=; an invalid
or unwritable target is reported and refused.

Normal interactive sessions and configured =-lc=/=-ic= commands load the
selected Quicklisp home's =local-init= files. Plain =-c=, scripts and safe mode
do not execute them, preserving their user-state-free behavior.

The flake installs an ordinary user command. It deliberately does not edit
=/etc/shells= or run =chsh=. A Nix profile symlink is not the attested,
root-owned stable path required by the login-shell installer below.

** Building from source

Source development is tested on Linux x86-64 with CCL 1.13. It requires Git,
Quicklisp, GNU make, coreutils, findutils, grep, awk, util-linux, m4, binutils,
and a C toolchain. Login installation also uses =flock=, =runuser= and
=getent=. Clone Clinedi and check out the locked revision:

#+begin_src sh
git clone https://github.com/luciusmagn/clinedi.git ~/common-lisp/clinedi
clinedi_commit=$(sed -n 's/^clinedi=//p' dependencies.lock)
git -C ~/common-lisp/clinedi checkout --detach "$clinedi_commit"
#+end_src

zoxide is optional. Install =fzf= as well when using its interactive =zi=
command.

Register the checkout's parent with Quicklisp, for example in
=~/quicklisp/local-init/common-lisp.lisp=:

#+begin_src lisp
(in-package #:quicklisp-client)
(pushnew (merge-pathnames #P"common-lisp/" (user-homedir-pathname))
         *local-project-directories*
         :test #'equal)
#+end_src

Set =CCLSH_QUICKLISP_SETUP= when =setup.lisp= is not at
=~/quicklisp/setup.lisp=. Every saved image includes Quicklisp and Clinedi.
The build refuses a missing Quicklisp setup, a non-Git Clinedi source, a dirty
checkout, a revision mismatch, or a different copy selected by Quicklisp.

Stock CCL is enough for regression checks and unsaved REPL development:

#+begin_src sh
scripts/check
#+end_src

#+begin_src lisp
(require :asdf)
(asdf:load-system "cclsh")
(cclsh:main)
#+end_src

A standalone saved image additionally requires a rebuilt CCL kernel containing
=ccl-cclsh-argv.patch=. The supported manual path applies both repository
patches to an official CCL 1.13 source checkout:

#+begin_src sh
git clone --branch v1.13 https://github.com/Clozure/ccl.git ../ccl
make ccl-kernel CCL_SOURCE=../ccl
make build CCL=../ccl/lx86cl64 \
  CCL_IMAGE=/usr/lib/ccl/lx86cl64.image
make install BINDIR="$HOME/.local/bin"
#+end_src

The boot image path is distribution-specific and must match the CCL source
generation. =ccl-kernel= patches the supplied checkout in place and leaves the
tracked changes for inspection. =ccl-cclsh-argv.patch= preserves cclsh command
and script operands. =ccl-linux-xstate.patch= validates and copies the complete
aligned Linux x86-64 signal state; see CCL issue 85 and upstream commit
=b1a59e443344f6f79d1069e1902de5d04c14e2e3= for the original diagnosis.

=make build= produces a matched =cclsh= kernel and =cclsh.image= in the
repository. It intentionally rejects stock CCL with
=kernel consumed a cclsh command operand=. Run =make ccl-kernel= and select
that rebuilt kernel instead. After a successful build, =make install= copies
the already validated pair. To rebuild and install in one user-local command,
pass the same kernel explicitly:

#+begin_src sh
CCLSH_CCL=../ccl/lx86cl64 \
CCLSH_CCL_IMAGE=/usr/lib/ccl/lx86cl64.image \
  scripts/install
#+end_src

** Root-owned login-shell installation

Run checks, build an attested pair, and preserve that attestation through the
integration checks:

#+begin_src sh
scripts/check
CCLSH_QUICKLISP_SETUP=/home/USER/quicklisp/setup.lisp \
  make login-build CCL_SOURCE=../ccl CCL=../ccl/lx86cl64 \
    CCL_IMAGE=/usr/lib/ccl/lx86cl64.image
make integration-check CCL=../ccl/lx86cl64 \
  CCL_IMAGE=/usr/lib/ccl/lx86cl64.image
sudo make install-login-shell LOGIN_USER=USER
#+end_src

=login-build= applies both patches, performs a clean CCL kernel build through
that exact checkout, publishes the saved image, and writes
=cclsh.attestation= over the patch and artifact hashes. Do not run a plain
=scripts/build= afterward because a non-login build removes the attestation.

=install-login-shell= validates the root-owned candidate as =LOGIN_USER=
before activation, switches a content-addressed release atomically, then adds
the stable path to =/etc/shells=. It refuses a missing or stale attestation and
never changes an account. Registration failure restores the previous release.
The target user must have a private primary group. One stable path serves one
login account; use another =BINDIR= for another account. =PREFIX=, =BINDIR=,
=SHELLS_FILE= and =LOGIN_USER= are Make variables. Custom system paths must
have canonical, root-owned parents that are not writable by non-root users.

Before probing configured mode, copy the sanitized starter configuration and
make it private:

#+begin_src sh
mkdir -p ~/.config/cclsh
cp examples/startup.lisp ~/.config/cclsh/startup.lisp
chmod 700 ~/.config/cclsh
chmod 600 ~/.config/cclsh/startup.lisp
#+end_src

Probe guaranteed behavior before changing the account. Test optional tools
such as zoxide separately when they are configured:

#+begin_src sh
sudo -u USER env HOME=/home/USER XDG_CONFIG_HOME=/home/USER/.config \
  SHELL=/usr/local/bin/cclsh /usr/local/bin/cclsh --version
sudo -u USER env HOME=/home/USER XDG_CONFIG_HOME=/home/USER/.config \
  SHELL=/usr/local/bin/cclsh CCLSH_SAFE=1 \
  /usr/local/bin/cclsh -c 'exit 0'
sudo -u USER env HOME=/home/USER XDG_CONFIG_HOME=/home/USER/.config \
  SHELL=/usr/local/bin/cclsh /usr/local/bin/cclsh -lc 'echo $PATH'
sudo chsh -s /usr/local/bin/cclsh USER
#+end_src

Keep another account, especially =root=, on a stock shell.

Run the regression checks with:

#+begin_src sh
scripts/check
#+end_src

After process, pipeline or terminal changes, rebuild the saved image and
run the PTY integration checks too. For an attested login build, use the
=login-build= and =make integration-check= sequence above:

#+begin_src sh
CCLSH_CCL=../ccl/lx86cl64 \
CCLSH_CCL_IMAGE=/usr/lib/ccl/lx86cl64.image \
  scripts/build
CCLSH_CCL=../ccl/lx86cl64 \
CCLSH_CCL_IMAGE=/usr/lib/ccl/lx86cl64.image \
  scripts/integration-check
#+end_src

* Behavior

- A line starting with =(= is evaluated as Lisp in the =cclsh-user=
  package, values are printed like a REPL.
- Otherwise the first word resolves to a shell command: a symbol whose
  value is a =command= instance runs as a builtin, anything else is
  looked up in =PATH= and run as an external program.
- When command lookup misses, a line containing only an explicit directory
  path changes into it without spelling =cd=: =..=, =./src=, =src/= and
  =~/work=. Builtins and executables still win, while a bare name such as
  =src= remains a command lookup.
- A lone word that names a bound variable, a keyword or a number
  evaluates REPL style when no command matches: =*last-status*= alone
  just prints its value.
- =~=, =$VAR=, =${VAR}= and =*=/=?= globs expand in bare words. Quoting
  with ="..."= (variables expand) and ='...'= (literal) is supported.
- Parens substitute Lisp values into commands: =echo (*balls*)= prints
  the variable, =echo (+ 1 2)= prints 3, lists splice, NIL vanishes.
- Unfinished input continues on the next line: unbalanced Lisp forms,
  open strings, and trailing backslashes.
- Undefined functions are reported with suggestions before argument
  evaluation can obscure them.
- =~/.config/cclsh/startup.lisp= is loaded for interactive sessions and
  configured one-shots, history persists in =~/.config/cclsh/history=
  (=XDG_CONFIG_HOME= is respected). cclsh keeps the configuration directory
  mode at =0700= and the history file at =0600= when it writes history.
- The built-in prompt shows =username@hostname=, the current Lisp package,
  directory and status sigil. Set =*prompt-function*= in =startup.lisp= to
  supply any other prompt policy or external renderer.
- External commands run as the terminal's foreground process group, so
  interactive programs work and Ctrl-C reaches the child, not the
  shell. Errors and Ctrl-C never drop you into the CCL debugger.
- Job control: =command &= starts a background job, Ctrl-Z stops the
  foreground command or pipeline, =jobs=, =fg= and =bg= manage them,
  and finished background jobs are announced before the next prompt.
  =exit= warns once about stopped jobs. Orderly exit sends SIGHUP to live
  jobs and SIGCONT to stopped jobs so logout does not leave them behind.
- =cclsh -c command=, script files, shebang scripts and piped input run
  without user state. Remote operations therefore work for executables
  already visible in sshd's environment, including ordinary system installs
  of scp, rsync and git; user-profile-only tools need an absolute path or a
  system-visible wrapper. Add =l= or =i= to command mode, such as
  =cclsh -lc command=, to load =startup.lisp= first. Short flags combine
  in any order. =CCLSH_SAFE=1= always skips startup.lisp and history.
- Scripts receive =*argv*= as a list containing the script path followed by
  every argument after it. Arguments are preserved literally and are not
  parsed as cclsh options. It is =nil= outside script mode. Use
  =cclsh -- -script.cclsh argument= for a dash-prefixed script path. Keep the
  executable basename =cclsh=, as the CCL kernel uses that name to recognize
  cclsh's argument boundary.
- Environment variables are lispy: =(setenv 'editor "vim")=,
  =(getenv :home)=, =(setf (env 'port) 8000)=, plus =export= and
  =unset= builtins. =CCLSH_PACKAGE= contains the canonical name of the
  current Lisp package and is refreshed before the configured prompt
  renderer and external commands launched through cclsh run.
- =(zoxide-setup)= enables =z=, =zi= and automatic recording of every
  successful =cd=. It is safe to call once from =startup.lisp= and does
  nothing destructive when zoxide is unavailable.

* Lisp helpers

#+begin_src lisp
;; Pipelines, sequencing, conditional chains, redirection
(cmd git "status")                ; one command, mid-Lisp
(pipe (ls "-la") (grep "lisp"))   ; ls -la | grep lisp
(seq (make "clean") (make))       ; make clean ; make
(all (make) (make "install"))     ; make && make install
(any (probe) (fallback))          ; probe || fallback
(pipe (make) (to "build.log"))    ; make > build.log, append-to appends
(pipe (from "in.txt") (wc "-l"))  ; wc -l < in.txt
(pipe (make) (error-to "err.log")) ; make 2> err.log
(capture (make) (merge-error))    ; both streams, like 2>&1 in $(...)
(capture (git "rev-parse" "HEAD")) ; $(git rev-parse HEAD) as a string

;; Run a program directly, returns the exit status
(run "git" "status")

;; Define your own commands (usable from the command line and Lisp)
(defcommand gs ()
  "Short git status."
  (run "git" "status" "--short"))

;; Optional ranked directory navigation
(zoxide-setup)
;; z project       ; query zoxide and cd to the result
;; zi              ; choose interactively through fzf
#+end_src

=*last-status*= holds the exit status of the last command.

* Line editor

The interactive editor is provided by Clinedi. Arrow keys navigate history and
the cursor. As you type, the newest
matching history entry appears after the cursor in dim text; Right or
C-f at the end accepts it. Multi-line history entries retain their
newlines. Alt-Enter, or Shift-Enter in terminals that report it, inserts an
explicit newline without submitting. Tab completes command names plus
directories at a slash-free command position, file paths in path-shaped
command words and arguments, and Lisp symbols. Command-position directories
keep the trailing =/= that makes them implicit =cd= paths. Supported keys:
C-a/C-e (home/end), Ctrl-Left/Right (move by
words), C-k/C-u (kill to end/whole line), C-w or C-Backspace/C-h (kill word),
C-l (clear screen), C-d (EOF on an empty line), C-c (abort line). Input is
highlighted with the standard 16 terminal colors: known commands green,
builtins and valid implicit directory paths cyan, unknown commands red,
strings yellow, comments and parens dim, keywords magenta, numbers cyan.
Output that ends without a newline gets a reverse video ⏎ marker, fish style.
The =help= builtin prints the built-in manual (=help substitution= etc.);
=docs/guide.org= is the long form, including login shell setup.
